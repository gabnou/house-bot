# HouseBot — Control Panel UI

Browser-based control panel for HouseBot. Built with SvelteKit + Skeleton UI (cerberus theme) + Tailwind v4.

## Tech stack

| | |
|---|---|
| Framework | SvelteKit v2 + Svelte 5 (runes mode) |
| UI library | Skeleton UI v4 (cerberus theme) |
| CSS | Tailwind v4 |
| Build adapter | `adapter-static` → `build/` |
| Dev server | Vite — `localhost:5252` |

## Pages

| Route | Status | Description |
|---|---|---|
| `/` | live | Home — logo, quick-link grid, dark/light toggle |
| `/status` | live | Health cards, config table, log viewer (4s auto-refresh) |
| `/admin` | live | Service control + Ollama model manager |
| `/prompts` | live | Per-skill LLM prompt editor (save / reset) |
| `/install` | live | Configuration wizard |

## Production build

The production build is generated by `housebot.sh` and served statically by FastAPI at `http://localhost:8000/`. You do not need to run a separate UI server in production.

```bash
# from the repo root
./housebot.sh ui-build
```

Rebuild whenever you pull UI source changes.

## Development server

```bash
# from the repo root
./housebot.sh ui-dev
```

Starts the Vite dev server at **http://localhost:5252** with HMR. API calls to `/admin/api/*` are proxied to the FastAPI bot at `:8000`, so the bot must be running alongside.

## Notes

- Svelte 5 runes mode only — use `$state()`, `$effect()`, `$derived()`, never `$:`
- Skeleton UI CSS is imported with **relative paths** in `src/app.css` (Tailwind v4 cannot resolve package `exports` globs)
- Dark mode uses the `.dark` class on `<html>`, toggled via `localStorage` key `colorMode`
- Git info (branch · hash · date) is baked into the bundle at build time via `vite.config.ts`
